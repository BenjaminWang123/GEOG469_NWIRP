About This Project

Mapping Community Impacts Across Washington

This platform is a county-level civic data and GIS reporting prototype created to help Northwest Immigrant Rights Project understand how immigration enforcement activity affects immigrant communities across Washington State.

What Is This Project About?

This project documents the broader community impacts connected to immigration enforcement activity. Instead of only mapping where enforcement activity happens, the platform focuses on how people’s daily lives may be affected afterward, including work, healthcare, school attendance, transportation, public space use, and community stability.

Community members can submit confidential, anonymized reports through the reporting portal. The system stores reports at the county level only, which helps protect privacy while still allowing NWIRP and community advocates to identify patterns across Washington State.

Project Goal

Build a privacy-first reporting and visualization platform that transforms community experiences into maps, dashboard summaries, and advocacy-ready insights.

Methodology

How We Approach the Project

The project combines participatory GIS, community reporting, web mapping, and database-driven visualization. The platform is designed around privacy, accessibility, and useful county-level aggregation rather than exact-location tracking.

01

Privacy-First Reporting

Users select a county or general area instead of submitting exact street addresses or home locations.

02

Impact Categorization

Each report is automatically categorized as economic, health, education, or social stability based on the selected incident type.

03

County-Level GIS

Washington county boundaries are used to support map interaction, county selection, aggregation, and choropleth visualization.

04

Analytics Dashboard

Reports are summarized by county, impact area, incident type, and time to identify broader community patterns.

Workflow

Project Workflow

1

User Selects a County

A user clicks a Washington county on the map, chooses from a county dropdown, or uses general location detection to estimate their county.

2

Report Form Is Completed

The user selects an incident type, optional date/time, and a general description. The form reminds users not to submit exact home addresses.

3

Backend Stores the Report

The frontend sends the report to an Express API, which inserts the record into a PostgreSQL database hosted through Supabase.

4

Data Becomes Visual

The submitted report appears in the map aggregation, blog page, and dashboard analytics after it is stored.

5

Patterns Support Advocacy

Aggregated results can help identify where communities are experiencing economic, health, educational, and social impacts.

Data Collection

What Data Is Collected?

The platform collects only the information needed for county-level analysis and community impact documentation. Reports should avoid personally identifying details or exact addresses.

Washington county or general area
Incident type selected by the user
Automatically assigned impact area
Optional event date and time
Optional general description
Optional image upload for future development

Technical Design

Platform Architecture

The platform uses a full-stack web mapping structure: static frontend pages, a Node/Express backend, a PostgreSQL database, and Washington county GeoJSON for GIS visualization.
